# Deployment Guide

This prototype uses a **single Dockerized service**:

- the React frontend is built into static assets
- the FastAPI backend serves the API and the frontend
- Tesseract is installed in the container image

Because of that architecture, you should deploy the **repo root** as one app service. You do **not** need separate frontend and backend deployments.

## Before you deploy

The repo is already prepared for hosting:

- root `Dockerfile` builds and runs the app
- `render.yaml` is included for Render
- health check endpoint: `/api/health`
- the container listens on the host-provided `PORT`

GitHub repo:

```text
https://github.com/mcooperstein/alcohol-label-verifier
```

## Fastest path: Render

Render is the easiest option for the current repo.

### Option A: Blueprint deployment

1. Sign in to Render
2. Click **New +**
3. Choose **Blueprint**
4. Connect GitHub if prompted
5. Select `mcooperstein/alcohol-label-verifier`
6. Render will detect `render.yaml`
7. Approve the service creation
8. Wait for the build and deploy to finish

### Option B: Manual web service

1. Sign in to Render
2. Click **New +**
3. Choose **Web Service**
4. Select `mcooperstein/alcohol-label-verifier`
5. Use these settings:

| Setting | Value |
| --- | --- |
| Runtime | Docker |
| Branch | `main` |
| Root directory | blank |
| Dockerfile path | `./Dockerfile` |
| Health check path | `/api/health` |

### After deploy

1. Open the generated service URL
2. Visit `/api/health`
3. Confirm it returns `{"status":"ok"}`
4. Open the root URL and test the UI

### Render notes

- No extra environment variables are required for this prototype
- If Render asks for a start command, leave it blank and use the Dockerfile default
- Free-tier services may cold start after inactivity

## Railway

Railway can deploy the same Docker-based repo directly.

1. Sign in to Railway
2. Create a **New Project**
3. Choose **Deploy from GitHub repo**
4. Select `mcooperstein/alcohol-label-verifier`
5. Let Railway build from the root `Dockerfile`
6. Wait for deploy completion
7. Open the generated Railway domain
8. Test `/api/health`

### Manual Railway settings if needed

| Setting | Value |
| --- | --- |
| Builder | Dockerfile |
| Dockerfile path | `Dockerfile` |
| Start command | leave blank |
| Port | leave blank and let Railway inject `PORT` |

### Railway notes

- The Dockerfile already respects `PORT`
- If deploy fails, confirm Railway is building from the repo root, not `frontend/` or `backend/`

## Fly.io

Fly works too, but it is more CLI-oriented.

### One-time setup

```bash
brew install flyctl
fly auth login
```

### Launch from the repo

```bash
cd /Users/mcooperstein/codingProjects/alcohol-label-verifier
fly launch --dockerfile Dockerfile --no-deploy
```

When prompted:

- choose an app name
- choose a region
- PostgreSQL: **No**
- Redis: **No**

Then deploy:

```bash
fly deploy
```

### Fly notes

- Fly typically creates a `fly.toml` config file
- Add an HTTP health check for `/api/health` if you want parity with Render

## Which URL should you submit?

Submit the public service URL generated by the host:

- Render: `https://your-service.onrender.com`
- Railway: `https://your-service.up.railway.app`
- Fly.io: `https://your-app.fly.dev`

That root URL should load the React UI.

## Recommended submission path

For the quickest working deployment:

1. Use **Render**
2. Deploy the GitHub repo as a **Blueprint**
3. Wait for the container build to complete
4. Confirm `/api/health`
5. Use the generated Render URL in your submission

## Troubleshooting

### The build succeeds but the app does not open

Make sure the host is deploying the **repo root** and using the **root Dockerfile**.

### The UI loads but review requests fail

Check container logs and confirm the process is running:

```text
uvicorn backend.app.main:app
```

### OCR fails after deployment

That usually means the deployment did not use the provided Dockerfile. The Docker image installs `tesseract-ocr`.

### The first request is slow

That is common on free tiers because the service may need to cold start.
